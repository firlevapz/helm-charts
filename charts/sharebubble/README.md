# ShareBubble Helm Chart

A Helm chart for deploying the ShareBubble Community Network application to Kubernetes.

## Prerequisites

- Kubernetes 1.23+
- Helm 3.8+
- PV provisioner support in the underlying infrastructure (for persistence)
- **Ingress mode:** an ingress controller installed (nginx, traefik, etc.)
- **Gateway API mode:** Gateway API CRDs (`gateway.networking.k8s.io/v1`) installed and a compatible controller running
- **cert-manager TLS (Gateway API only):** cert-manager >= 1.15 with the Gateway API feature gate enabled

## Installing the Chart

```bash
helm install sharebubble ./charts/sharebubble -f my-values.yaml
```

Minimum required values:

```bash
helm install sharebubble ./charts/sharebubble \
  --set backend.secrets.secretKey="your-secret-key" \
  --set postgresql.auth.password="db-password" \
  --set postgresql.auth.postgresPassword="postgres-password"
```

## Traffic Routing

Both ingress and Gateway API modes route traffic identically, mirroring the built-in nginx config:

| Path prefix | Destination |
| ----------------------- | ---------------------- |
| `/api`, `/accounts`, `/admin` | Backend service (Django) |
| everything else (`/`) | Frontend service (nginx/SPA) |

Only **one** of `frontend.ingress` or `frontend.gateway` + `frontend.httpRoute` should be enabled at a time.

### Ingress (default)

Standard `networking.k8s.io/v1` Ingress. Works with any ingress controller.
Enabled by default; disable it to switch to Gateway API.

| Parameter | Description | Default |
| -------------------------------------- | ----------------------------------------------------- | ----------------------- |
| `frontend.ingress.enabled` | Enable Ingress resource | `true` |
| `frontend.ingress.className` | IngressClass name. Empty = cluster default. | `""` |
| `frontend.ingress.annotations` | Annotations added to the Ingress | `{}` |
| `frontend.ingress.hosts` | List of `{host, paths[]}` entries | `[{host: sharebubble.local, paths: [{path: /, pathType: Prefix}]}]` |
| `frontend.ingress.tls` | TLS configuration blocks | `[]` |

**Routing generated by the chart** (for every host):
- `Prefix /api`, `Prefix /accounts`, `Prefix /admin` → backend service
- `Prefix /` → frontend service

### Gateway API

Gateway API resources (`Gateway` + `HTTPRoute`). Requires `frontend.ingress.enabled=false`.

| Parameter | Description | Default |
| ------------------------------------------- | ----------------------------------------------------- | ----------------------- |
| `frontend.gateway.enabled` | Create a `Gateway` resource | `false` |
| `frontend.gateway.gatewayClassName` | GatewayClass name. Empty = cluster default. | `""` |
| `frontend.gateway.annotations` | Annotations added to the Gateway | `{}` |
| `frontend.gateway.labels` | Extra labels added to the Gateway | `{}` |
| `frontend.gateway.addresses` | Gateway address overrides | `[]` |
| `frontend.gateway.listeners` | Listener definitions | see values.yaml |
| `frontend.httpRoute.enabled` | Create an `HTTPRoute` resource | `false` |
| `frontend.httpRoute.annotations` | Annotations added to the HTTPRoute | `{}` |
| `frontend.httpRoute.parentRefs` | Explicit parentRefs. Auto-derived when empty. | `[]` |
| `frontend.httpRoute.hostnames` | Hostnames matched by the HTTPRoute | `[sharebubble.local]` |
| `frontend.httpRoute.backendPrefixes` | Path prefixes forwarded to the backend service | `[/api, /accounts, /admin]` |
| `frontend.httpRoute.frontendPrefix` | Catch-all prefix served by the frontend service | `/` |
| `frontend.httpRoute.additionalRules` | Extra HTTPRoute rules appended verbatim | `[]` |

When `frontend.httpRoute.parentRefs` is empty and `frontend.gateway.enabled=true` the chart
automatically derives the parentRef from the first configured listener.

### cert-manager TLS (Gateway API only)

Integrates with [cert-manager's Gateway API support](https://cert-manager.io/docs/usage/gateway/).
Only applies when `frontend.gateway.enabled=true`.

When enabled the chart:
1. Annotates the Gateway with `cert-manager.io/cluster-issuer` (or `cert-manager.io/issuer`).
2. Auto-generates one HTTPS listener on port 443 per hostname in `frontend.httpRoute.hostnames`.
3. Optionally adds an HTTP listener on port 80 with a companion `HTTPRoute` that issues a 301 redirect to HTTPS.

| Parameter | Description | Default |
| ------------------------------------------ | ------------------------------------------------------- | ----------------------- |
| `frontend.certManager.enabled` | Enable cert-manager TLS provisioning | `false` |
| `frontend.certManager.clusterIssuerName` | `ClusterIssuer` name to use | `letsencrypt-prod` |
| `frontend.certManager.issuerName` | Namespace-scoped `Issuer` name (overrides clusterIssuer) | `""` |
| `frontend.certManager.secretName` | TLS Secret name cert-manager will write. Defaults to `<fullname>-tls`. | `""` |
| `frontend.certManager.httpRedirect` | Add HTTP→HTTPS 301 redirect listener + HTTPRoute | `true` |

## Examples

### Default — Ingress (nginx)

```yaml
# values-ingress.yaml
frontend:
  ingress:
    enabled: true
    className: nginx
    annotations:
      cert-manager.io/cluster-issuer: letsencrypt-prod
    hosts:
      - host: app.example.com
        paths:
          - path: /
            pathType: Prefix
    tls:
      - secretName: sharebubble-tls
        hosts:
          - app.example.com

backend:
  django:
    allowedHosts: app.example.com
  secrets:
    secretKey: your-very-secret-key-here

postgresql:
  auth:
    password: secure-db-password
    postgresPassword: secure-postgres-password
```

### Gateway API — plain HTTP (local/dev)

```yaml
# values-gateway-http.yaml
frontend:
  ingress:
    enabled: false

  gateway:
    enabled: true
    listeners:
      - name: web
        protocol: HTTP
        port: 80
        hostname: app.example.com
        allowedRoutes:
          namespaces:
            from: Same

    enabled: true
    hostnames:
      - app.example.com

    allowedHosts: app.example.com
  secrets:
    secretKey: your-very-secret-key-here

postgresql:
  auth:
    password: secure-db-password
    postgresPassword: secure-postgres-password
```

### Gateway API — HTTPS with cert-manager (recommended for production)

cert-manager automatically provisions and renews the TLS certificate.

```yaml
# values-gateway-tls.yaml
frontend:
  ingress:
    enabled: false

  gateway:
    enabled: true

  certManager:
    enabled: true
    clusterIssuerName: letsencrypt-prod   # must exist in the cluster
    httpRedirect: true                    # HTTP → HTTPS 301 redirect

  httpRoute:
    enabled: true
    hostnames:
      - app.example.com

backend:
  django:
    allowedHosts: app.example.com
  secrets:
    secretKey: your-very-secret-key-here

postgresql:
  auth:
    password: secure-db-password
    postgresPassword: secure-postgres-password
```

This produces:
- A **Gateway** annotated with `cert-manager.io/cluster-issuer: letsencrypt-prod`
  - HTTPS listener on port 443 (TLS terminated, cert managed by cert-manager)
  - HTTP listener on port 80
- An **HTTPRoute** (HTTPS) routing `/api`, `/accounts`, `/admin` → backend; everything else → frontend
- An **HTTPRoute** (HTTP) redirecting all traffic to HTTPS with a 301

### Gateway API — manual TLS Secret (pre-existing cert)

```yaml
# values-gateway-manual-tls.yaml
frontend:
  ingress:
    enabled: false

  gateway:
    enabled: true
    listeners:
      - name: https
        protocol: HTTPS
        port: 443
        hostname: app.example.com
        tls:
          mode: Terminate
          certificateRefs:
            - kind: Secret
              group: ""
              name: my-tls-secret
        allowedRoutes:
          namespaces:
            from: Same
      - name: http
        protocol: HTTP
        port: 80
        allowedRoutes:
          namespaces:
            from: Same

  httpRoute:
    enabled: true
    hostnames:
      - app.example.com

backend:
  django:
    allowedHosts: app.example.com
  secrets:
    secretKey: your-very-secret-key-here

postgresql:
  auth:
    password: secure-db-password
    postgresPassword: secure-postgres-password
```

## Upgrading

```bash
helm upgrade sharebubble ./charts/sharebubble -f my-values.yaml
```

## Uninstalling

```bash
helm uninstall sharebubble
```

PVCs are not deleted automatically. To remove them:

```bash
kubectl delete pvc -l app.kubernetes.io/instance=sharebubble
```
